    ## Introduction

    This guide provides comprehensive instructions for integrating the multi-chain architecture documentation with Docusaurus. It covers the setup process, configuration options, and best practices for a successful integration.

    <Admonition type="info">
      Docusaurus is a modern static website generator that builds optimized documentation sites with React. It provides features like versioning, search, and internationalization out of the box.
    </Admonition>

    ## Prerequisites

    Before beginning the integration process, ensure you have:

    - Node.js 16.14 or higher installed
    - Yarn package manager (recommended) or npm
    - Git for version control
    - Basic knowledge of React and Markdown
    - Access to the documentation repository

    ## Installation Process

    ### Step 1: Create a New Docusaurus Project

    Start by creating a new Docusaurus project:

    <CodeBlock language="bash">
    {`npx create-docusaurus@latest multi-chain-docs classic --typescript`}
    </CodeBlock>

    This command creates a new Docusaurus project with the classic template and TypeScript support.

    ### Step 2: Configure Project Structure

    Navigate to the project directory and configure the basic structure:

    <CodeBlock language="bash">
    {`cd multi-chain-docs
mkdir -p docs/overview docs/architecture docs/finance docs/tokenomics
mkdir -p src/components/docs src/components/common`}
    </CodeBlock>

    ## Component Migration

    ### Step 1: Create Base Components

    Create the base components in the `src/components/docs` directory:

    <CodeBlock language="typescript">
    {`// src/components/docs/SectionLevelOne.tsx
import React, { ReactNode } from 'react';
import styles from './styles.module.css';

interface SectionLevelOneProps {
  id: string;
  title: string;
  children: ReactNode;
}

export function SectionLevelOne({ id, title, children }: SectionLevelOneProps) {
  return (
    <section id={id} className={styles.sectionLevelOne}>
      <h1 className={styles.sectionTitle}>{title}</h1>
      <div className={styles.sectionContent}>{children}</div>
    </section>
  );
}`}
    </CodeBlock>

    ### Step 2: Create Content Components

    Create the content components in the same directory:

    <CodeBlock language="typescript">
    {`// src/components/docs/ContentBlock.tsx
import React, { ReactNode } from 'react';
import styles from './styles.module.css';

interface ContentBlockProps {
  children: ReactNode;
}

export function ContentBlock({ children }: ContentBlockProps) {
  return (
    <div className={styles.contentBlock}>
      {children}
    </div>
  );
}`}
    </CodeBlock>

    ### Step 3: Create Utility Components

    Create utility components for diagrams, code blocks, etc.:

    <CodeBlock language="typescript">
    {`// src/components/docs/DiagramBlock.tsx
import React, { ReactNode } from 'react';
import styles from './styles.module.css';

interface DiagramBlockProps {
  children: ReactNode;
  caption?: string;
  id?: string;
}

export function DiagramBlock({ children, caption, id }: DiagramBlockProps) {
  return (
    <figure id={id} className={styles.diagramBlock}>
      <div className={styles.diagramContent}>{children}</div>
      {caption && <figcaption className={styles.diagramCaption}>{caption}</figcaption>}
    </figure>
  );
}`}
    </CodeBlock>

    ## Content Migration

    ### Step 1: Convert TSX to MDX

    Convert existing TSX components to MDX format:

    1. Extract content from TSX components
    2. Create corresponding MDX files
    3. Add appropriate frontmatter
    4. Import and use the Docusaurus components

    ### Step 2: Migrate Images and Assets

    Move images and assets to the `static` directory:

    <CodeBlock language="bash">
    {`mkdir -p static/img/diagrams static/img/icons
cp -r public/diagrams/* static/img/diagrams/
cp -r public/*.png static/img/`}
    </CodeBlock>

    Update image references in MDX files:

    <CodeBlock language="markdown">
    {`// Before
<img src="/diagrams/network-topology.png" alt="Network Topology" />

// After
<img src="/img/diagrams/network-topology.png" alt="Network Topology" />`}
    </CodeBlock>

    ### Step 3: Configure Metadata

    Add appropriate frontmatter to all MDX files:

    <CodeBlock language="markdown">
    {`---
title: Document Title
description: Document description for SEO
sidebar_label: Shorter Title for Sidebar
sidebar_position: 1
keywords: ['keyword1', 'keyword2']
---`}
    </CodeBlock>

    ## Plugin Configuration

    ### Search Configuration

    Configure the search plugin for optimal documentation search:

    <CodeBlock language="javascript">
    {`// In docusaurus.config.js
themeConfig: {
  // ...
  algolia: {
    appId: 'YOUR_APP_ID',
    apiKey: 'YOUR_API_KEY',
    indexName: 'multi-chain-architecture',
    contextualSearch: true,
    searchParameters: {},
  },
}`}
    </CodeBlock>

    ### Versioning Configuration

    Set up versioning for documentation:

    <CodeBlock language="bash">
    {`yarn docusaurus docs:version 1.0.0`}
    </CodeBlock>

    Configure versioning in `docusaurus.config.js`:

    <CodeBlock language="javascript">
    {`// In docusaurus.config.js
themeConfig: {
  // ...
  versions: {
    current: {
      label: 'Current',
      path: '',
    },
  },
}`}
    </CodeBlock>

    <CodeBlock language="javascript">

    Configure for Vercel deployment:

    <CodeBlock language="json">
    {`{
  "buildCommand": "yarn build",
  "outputDirectory": "build",
  "framework": "docusaurus",
  "github": {
    "silent": true
  }
}`}
    </CodeBlock>
